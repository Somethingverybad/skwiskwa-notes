# Руководство по внесению изменений

Спасибо за интерес к проекту! Мы рады любым улучшениям.

## Как внести изменения

1. **Fork проекта**
   ```bash
   git clone https://github.com/your-username/notion-clone.git
   cd notion-clone
   ```

2. **Создайте ветку для изменений**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Внесите изменения**
   - Следуйте стилю кода проекта
   - Добавьте комментарии для сложной логики
   - Обновите документацию при необходимости

4. **Протестируйте изменения**
   - Убедитесь, что backend работает: `python manage.py test`
   - Проверьте frontend: `npm run build`
   - Протестируйте функциональность вручную

5. **Commit и Push**
   ```bash
   git add .
   git commit -m "Описание изменений"
   git push origin feature/your-feature-name
   ```

6. **Создайте Pull Request**
   - Опишите, что вы изменили и почему
   - Добавьте скриншоты для UI изменений
   - Укажите связанные issues

## Стиль кода

### Python (Backend)
- Следуйте PEP 8
- Используйте type hints где возможно
- Добавляйте docstrings для функций и классов

### TypeScript (Frontend)
- Используйте функциональные компоненты
- Предпочитайте TypeScript типы вместо any
- Именуйте файлы в PascalCase для компонентов
